# Design RFCs

Design RFCs for `pss-runtime` live here as numbered, versioned documents. An RFC captures a proposed direction — status, motivation, proposed surface, implementation plan, and open questions — before code lands. RFCs are migrated into the repo (rather than living only in the issue tracker) so they version with the codebase and stay discoverable from a checkout.
